# 🍻 원광대학교 사회과학대학 주점 테이블 관리 시스템

안녕하세요! 이 프로젝트는 대학교 축제의 꽃, 주점을 더욱 스마트하고 효율적으로 운영하기 위해 탄생한 **실시간 테이블 관리 시스템**입니다.

손님들이 테이블의 QR 코드를 스캔하여 직접 이용 정보를 등록하면, 관리자는 모든 테이블 현황을 스마트폰이나 PC로 한눈에 파악하고 관리할 수 있습니다. 더 이상 수기로 테이블 정보를 관리하거나, 남은 시간을 일일이 계산할 필요가 없습니다.

## ✨ 한눈에 보는 주요 기능

| 기능 | 설명 |
| :--- | :--- |
| **📲 QR코드 기반 등록** | 손님은 테이블의 QR 코드를 스캔하여 복잡한 절차 없이 **테이블 번호, 이름, 연락처, 인원**만으로 빠르게 이용 등록을 할 수 있습니다. |
| **📊 실시간 대시보드** | 관리자는 현재 사용 중인 모든 테이블의 현황과 **남은 이용 시간**을 실시간으로 확인할 수 있습니다. 대시보드는 남은 시간이 적은 순으로 자동 정렬되어, 곧 비게 될 테이블을 쉽게 예측하고 다음 손님을 맞이할 준비를 할 수 있습니다. |
| **✏️ 유연한 정보 관리** | 손님이 정보를 잘못 입력했나요? 걱정 마세요! 관리자는 언제든지 **테이블 번호, 이름, 연락처, 인원** 정보를 수정할 수 있습니다. 손님이 퇴장하면 **삭제** 버튼 하나로 간단하게 테이블을 정리할 수 있습니다. |
| **🔔 자동 알림 시스템** | 테이블 이용 시간이 **10분, 5분 남았거나 만료**되면 관리자 페이지에 자동으로 알림이 표시됩니다. 이를 통해 손님에게 미리 안내하여 원활한 테이블 회전을 유도할 수 있습니다. |

## 📖 이렇게 사용해요! (사용 시나리오)

### 👨‍👩‍👧‍👦 손님의 경우

1.  주점 테이블에 앉아 QR 코드를 스캔하여 **손님 페이지**([https://wkucollegeofsocialscience.up.railway.app/](https://wkucollegeofsocialscience.up.railway.app/))에 접속합니다.
2.  안내에 따라 **테이블 번호, 대표자 이름, 전화번호, 총인원**을 입력하고 '등록하기' 버튼을 누릅니다.
3.  "감사합니다!" 메시지와 함께 남은 이용 시간을 확인할 수 있는 **개인 타이머 페이지**로 이동합니다.
4.  이용 시간이 끝나면 다음 손님을 위해 자리를 정리합니다.

### 👮 관리자의 경우

1.  **관리자 페이지**([https://wkucollegeofsocialscience.up.railway.app/#admin](https://wkucollegeofsocialscience.up.railway.app/#admin))에 접속하여 전체 테이블 현황을 확인합니다.
2.  (상황 1) 빈자리를 문의하는 손님이 오면, 대시보드에서 곧 시간이 만료되는 테이블(남은 시간이 적거나, 빨간색으로 표시된 테이블)을 확인하고 안내합니다.
3.  (상황 2) 손님이 인원을 잘못 입력했다고 알려주면, 해당 테이블의 **`수정`** 버튼을 눌러 팝업 창에서 인원 정보를 올바르게 고쳐주고 **`저장`** 합니다.
4.  (상황 3) 이용 시간이 만료된 손님이 퇴장하면, 해당 테이블의 **`삭제`** 버튼을 눌러 목록에서 제거하고 자리를 정리합니다.
5.  (상황 4) 브라우저 상단에 알람이 뜨면 이용 시간이 얼마 남지 않은 테이블을 확인하고 손님에게 미리 알려줄 준비를 합니다.

## 🛠️ 기술 스택

- **Frontend**: React, TypeScript
- **Backend**: Node.js, Express
- **Real-time Communication**: Socket.IO
- **Deployment**: Docker, Railway

## 🚀 로컬 개발 환경 설정

### 1. 사전 준비

- [Node.js](https://nodejs.org/) (v20.x 이상)가 설치되어 있어야 합니다.
- 프로젝트를 클론합니다.
  ```bash
  git clone https://github.com/nongnola1221/WKU_Festival_Bar_Management.git
  cd WKU_Festival_Bar_Management
  ```

### 2. 의존성 설치

```bash
# 루트, 클라이언트, 서버 의존성 동시 설치
npm install && npm install --prefix client && npm install --prefix server
```

### 3. 애플리케이션 실행 (개발 모드)

두 개의 터미널을 열고 각각 다음 명령어를 실행합니다.

- **터미널 1: 클라이언트 실행** (`http://localhost:3000`)
  ```bash
  npm start --prefix client
  ```

- **터미널 2: 서버 실행** (`http://localhost:4000`)
  ```bash
  npm run server
  ```

## 🚢 배포

이 프로젝트는 `Dockerfile`을 사용하여 컨테이너화되며, [Railway](https://railway.app/)를 통해 배포됩니다.

- **빌드 및 배포**: `main` 브랜치에 코드가 푸시되면, Railway는 `Dockerfile`을 사용하여 자동으로 이미지를 빌드하고 배포를 진행합니다.
- **데이터 영속성**: 현재는 파일 시스템(`server/db.json`)을 사용하여 데이터를 저장합니다.
- **무중단 배포**: Railway의 헬스 체크 기능을 사용하여 무중단 배포가 설정되어 있습니다. (`/health` 엔드포인트 사용)
