# 🍻 원광대학교 사회과학대학 주점 테이블 관리 시스템

안녕하세요! 축제 주점 운영을 도와줄 똑똑한 비서, **실시간 테이블 관리 시스템**에 오신 것을 환영합니다.

## 🤔 이 앱은 무엇인가요?

축제 때마다 종이에 적고, 시간을 일일이 확인하며 테이블을 관리하기 힘드셨죠? 이 시스템은 그런 불편함을 해결하기 위해 태어났습니다.

-   **손님은** 테이블의 QR 코드를 찍어 간편하게 이용 등록을 하고,
-   **관리자는** 스마트폰이나 PC로 모든 테이블 현황을 한눈에 보며 손쉽게 관리할 수 있습니다.

마치 식당의 '웨이팅 시스템'처럼, 우리 주점만의 스마트한 테이블 관리 시스템이라고 생각하시면 됩니다.

---

## ✨ 한눈에 보는 주요 기능

| 아이콘 | 기능 | 설명 |
| :--- | :--- | :--- |
| **📲** | **QR코드로 간편 등록** | 손님은 테이블의 QR 코드를 스캔하여 복잡한 절차 없이 **테이블 번호, 이름, 연락처, 인원**만으로 2분 안에 이용 등록을 마칠 수 있습니다. |
| **📊** | **실시간 상황판** | 관리자는 현재 어떤 테이블이 사용 중인지, 각 테이블의 **남은 시간**이 얼마인지 실시간으로 확인할 수 있습니다. 대시보드는 남은 시간이 적은 순으로 자동 정렬되어, 곧 비게 될 테이블을 쉽게 예측하고 다음 손님을 맞이할 준비를 할 수 있습니다. |
| **✏️** | **간편한 정보 관리** | 손님이 정보를 잘못 입력했어도 괜찮아요! 관리자는 언제든지 **테이블 번호, 이름, 연락처, 인원** 정보를 수정할 수 있습니다. 손님이 퇴장하면 **삭제** 버튼 하나로 간단하게 테이블을 정리할 수 있습니다. |
| **🔔** | **똑똑한 자동 알림** | 테이블 이용 시간이 **10분, 5분 남았거나 만료**되면 **브라우저 상단에 잠시 알림이 표시됩니다.** 이를 통해 손님에게 미리 안내하여 원활한 테이블 회전을 유도할 수 있습니다. (최초 접속 시 알림 권한을 허용해주세요!) |

## 📖 쉽고 빠른 사용 방법

### 👨‍👩‍👧‍👦 손님이라면?

1.  테이블에 앉아 QR 코드를 휴대폰 카메라로 스캔하세요.
2.  아래 링크와 같은 등록 페이지가 열립니다.
    > **손님용 등록 페이지**<br>
    > [https://wkucollegeofsocialscience.up.railway.app/](https://wkucollegeofsocialscience.up.railway.app/)
3.  안내에 따라 **테이블 번호, 대표자 이름, 전화번호, 총인원**을 입력하고 **'등록하기'** 버튼을 누르세요.
4.  "감사합니다!" 메시지와 함께 남은 이용 시간을 보여주는 개인 타이머 페이지로 이동하면 등록 완료!

### 👮 관리자라면?

1.  아래 링크를 클릭하거나, 주소창에 직접 입력하여 관리자 페이지에 접속하세요.
    > **관리자용 페이지**<br>
    > [https://wkucollegeofsocialscience.up.railway.app/#admin](https://wkucollegeofsocialscience.up.railway.app/#admin)
2.  **(손님 맞이)** 빈자리를 문의하는 손님이 오면, 대시보드에서 곧 시간이 만료되는 테이블(남은 시간이 적거나, 빨간색으로 표시된 테이블)을 확인하고 안내합니다.
3.  **(정보 수정)** 손님이 정보를 잘못 입력했다면, 해당 테이블의 **`수정`** 버튼을 눌러 팝업 창에서 정보를 고쳐주고 **`저장`** 버튼을 누릅니다.
4.  **(테이블 정리)** 이용이 끝난 손님이 퇴장하면, 해당 테이블의 **`삭제`** 버튼을 눌러 목록에서 깔끔하게 지웁니다.

### 기술 스택

- **Frontend**: React, TypeScript
- **Backend**: Node.js, Express
- **Real-time Communication**: Socket.IO
- **Deployment**: Docker, Railway

### 로컬 개발 환경 설정

#### 1. 사전 준비

- [Node.js](https://nodejs.org/) (v20.x 이상)가 설치되어 있어야 합니다.
- 프로젝트를 클론합니다.
  ```bash
  git clone https://github.com/nongnola1221/WKU_Festival_Bar_Management.git
  cd WKU_Festival_Bar_Management
  ```

#### 2. 의존성 설치

```bash
# 루트, 클라이언트, 서버 의존성 동시 설치
npm install && npm install --prefix client && npm install --prefix server
```

#### 3. 애플리케이션 실행 (개발 모드)

두 개의 터미널을 열고 각각 다음 명령어를 실행합니다.

- **터미널 1: 클라이언트 실행** (`http://localhost:3000`)
  ```bash
  npm start --prefix client
  ```

- **터미널 2: 서버 실행** (`http://localhost:4000`)
  ```bash
  npm run server
  ```

### 배포

이 프로젝트는 `Dockerfile`을 사용하여 컨테이너화되며, [Railway](https://railway.app/)를 통해 배포됩니다.

- **빌드 및 배포**: `main` 브랜치에 코드가 푸시되면, Railway는 `Dockerfile`을 사용하여 자동으로 이미지를 빌드하고 배포를 진행합니다.
- **데이터 영속성**: 현재는 파일 시스템(`server/db.json`)을 사용하여 데이터를 저장합니다.
- **무중단 배포**: Railway의 헬스 체크 기능을 사용하여 무중단 배포가 설정되어 있습니다. (`/health` 엔드포인트 사용)
